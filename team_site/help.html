Execution Help Center

Step-by-step guide for running Phase 6 (10-repo soak) and Phase 7 (org-wide rollout).

← Back to Requests Hub

Phase 6 — 10-Repo Soak (Nov 16–30)

  1. Select repos: follow criteria in `planning/60_launch/launch_runbook.md` §2.1.
  2. Prep checklists: run Pre-Rollout Checklist (§1) per repo (infra, config, training).
  3. Onboard teams: invite to training sessions (`planning/60_launch/training/`).
  4. Daily standup: 9 AM PT with Platform + QA + SRE (log notes in runbook §2.4).
  5. Monitor dashboards: Grafana links in `planning/50_pilot/evidence/data/dashboards_summary.md`.
  6. Capture evidence:
    • QA metrics → `planning/50_pilot/evidence/qa/template_metrics.md`
    • SRE logs → `planning/50_pilot/evidence/sre/soak_test_results.md`
    • Engineering run logs → `planning/50_pilot/evidence/qa/flowengine_run_logs.md`

Phase 7 — Org-Wide Rollout (Dec 9–20)

  1. Confirm G6 GO: update `planning/approvals/G6_signoff.md` with final evidence.
  2. Wave assignments: publish schedule (Wave 1: Dec 9–13, Wave 2: Dec 16–20).
  3. Run enablement: deliver modules per `planning/60_launch/training/enablement_plan.md`.
  4. Go-live day: follow hour-by-hour plan (runbook §4) and post Slack updates.
  5. Monitor & alert: use runbook §5 thresholds; escalate via P1/P2/P3 playbooks.
  6. Rollback when needed: choose scenario in runbook §6 (pilot, previous version, manual review).
  7. Post-launch support: host office hours (runbook §7) and track incidents (runbook §8).
